# Colabora Documentation

This directory contains all project documentation organized by status and relevance.

## Directory Structure

- **`active/`** - Current, actively maintained documentation
  - Guides for developers and users
  - Current architecture and patterns
  - Deployment and setup instructions

- **`archive/`** - Historical documentation
  - Completed implementation plans
  - Old strategy documents
  - Historical analysis and planning docs

## Key Documents

### For Developers

- **[ARCHITECTURE.md](ARCHITECTURE.md)** - System architecture overview
- **[PATTERNS.md](PATTERNS.md)** - Code patterns and conventions
- **[active/CODEBASE_SUMMARY.md](active/CODEBASE_SUMMARY.md)** - Current codebase status and issues

### For Users

- **[active/TESTING_GUIDE.md](active/TESTING_GUIDE.md)** - Testing guide

## Documentation Status

- **Active** - Current, up-to-date documentation
- **Archive** - Historical reference only, may be outdated

## Contributing

When adding new documentation:

1. **Active docs** - Place in `active/` if it's current and relevant
2. **Completed work** - Move to `archive/` when implementation is complete
3. **Root directory** - Keep only `QUICK_START.md` and `README.md` in root

## Finding Documentation

- **Current architecture?** → `ARCHITECTURE.md`
- **Code patterns?** → `PATTERNS.md`
- **How to use?** → `QUICK_START.md` or `active/USAGE_GUIDE.md`
- **How to deploy?** → `active/DEPLOYMENT_GUIDE.md`
- **Historical context?** → `archive/`
