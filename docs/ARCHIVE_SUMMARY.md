## Archive Policy

### When to Archive

Files should be archived when:
- They are superseded by newer documentation
- Issues documented have been resolved
- Analysis reports are outdated
- Implementation plans have been completed
- They are no longer relevant to current development

### Archive Organization

- Group related files by date in subdirectories
- Use descriptive directory names (e.g., `2025-01-evaluation-update`)
- Include README.md in archive directories explaining contents
- Preserve all files for historical reference
